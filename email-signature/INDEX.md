## üí° Pro Tips

### Tip #1: Preview First
Always open `preview.html` before reading documentation. Visual context helps understanding.

### Tip #2: Bookmark Three Files
- `DEPLOYMENT_GUIDE.md` - Your main reference
- `TEMPLATE_VARIABLES.md` - For variable issues
- `preview.html` - For visual comparison

### Tip #3: Print the Checklist
Print the testing checklist from `DEPLOYMENT_GUIDE.md` and check off as you test each client.

### Tip #4: Start with Static
Unless you have specific reasons for animated version, deploy static first. You can always upgrade later.

### Tip #5: Test Logo First
Before doing anything else, host the logo and verify it's publicly accessible. This is the most common issue.

## ‚ùì Quick FAQ

**Q: Which file do I deploy?**
A: `signature-static.html` (recommended) or `signature-animated.html` (optional)

**Q: Where do I see what it looks like?**
A: Open `preview.html` in any web browser

**Q: How do I deploy to Microsoft 365?**
A: Follow `DEPLOYMENT_GUIDE.md` step-by-step

**Q: What variables do I need to replace?**
A: See `TEMPLATE_VARIABLES.md` for complete mapping

**Q: Should I use static or animated?**
A: Read `VERSION_COMPARISON.md` - we recommend static for most organizations

**Q: Where is the logo?**
A: `../assets/images/SYNTRA REFINING WHITE LOGO KIT - SBG.png` - you need to host it publicly

**Q: What if something breaks?**
A: Check Troubleshooting section in `DEPLOYMENT_GUIDE.md`

**Q: Can users customize their signature?**
A: No - this is a centrally-managed, mandatory signature
